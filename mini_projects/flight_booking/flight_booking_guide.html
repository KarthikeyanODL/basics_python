✈ Flight Booking App

A simple Flask web application — Step-by-step setup guide

Python 3 Flask HTML Windows & Mac
📋 What This App Does

This is a beginner-friendly web application built with Python + Flask. It lets a user:

  • Search for a flight by entering source and destination city
  • View flight details — seats available and price
  • Book one or more seats
  • See a booking confirmation with total amount

📦 Check pip (Python package installer)

pip comes with Python automatically. But should you type pip or pip3?

pip vs pip3 — which one to use?

Both install the same thing. The difference is which Python version they point to:

  • pip  → may point to Python 2 on older Mac systems
  • pip3 → always points to Python 3

Simple rule:
  • Windows → use pip
  • Mac    → use pip3 (safer choice)

Not sure? Run both commands below and use whichever shows (python 3.x) at the end.

Run this to check:

pip --version

Expected output — look for python 3 at the end:

pip 24.0 from C:\Python312\lib\site-packages\pip (python 3.12)
                                                             ^^^^^^^^^^^^ this confirms it's Python 3
✅ On Windows, pip almost always points to Python 3. Use pip.

Run both and check which one shows python 3:

pip --version
pip3 --version

Look for python 3 at the end of the output:

pip3 24.0 from /usr/local/lib/python3.12/... (python 3.12)
                                                            ^^^^^^^^^^^^ use this one
✅ On Mac, always prefer pip3 — it guarantees Python 3.

What is 127.0.0.1:5000?
127.0.0.1 means "this computer" (localhost). 5000 is the port Flask uses by default. The app runs only on your computer — it is not on the internet.

🧠 How the Code Works
🔗 The 3 Flask routes explained
URLMethodWhat it doesTemplate
/GETShows the search formsearch.html
/searchPOSTLooks up flight in dictionary, shows detailsflight.html
/bookPOSTBooks seats, updates dictionary, shows confirmationsuccess.html
GET vs POST:
GET = just loading/viewing a page (no data sent).
POST = submitting a form with data (source city, seats, etc.).
🔁 How data travels between pages

Data moves between pages using HTML forms. There are two types of inputs used:

1. Visible input — the user types into it:

<input type="text" name="source">

2. Hidden input — invisible to the user, carries data silently:

<input type="hidden" name="flight_no" value="{{ flight_no }}">

This is how flight.html sends the flight number to the /book route without the user seeing it.

Flask reads form data using:

flight_no = request.form["flight_no"]
seats     = int(request.form["seats"])
📊 How seats are updated

The flights dictionary lives in memory while the app is running. When a booking is confirmed:

# Subtract the booked seats from available seats
details["seats"] -= seats

# Calculate total cost
total = seats * details["price"]
⚠️ Note for students: The dictionary resets every time you restart the app (python app.py), because data is stored in memory, not in a database. This is normal for a learning project.
🛠 Troubleshooting
❌ "ModuleNotFoundError: No module named 'flask'"

Flask is not installed. Run this command and try again:

pip install flask
pip3 install flask
❌ "python is not recognized" (Windows)

Python was installed but not added to PATH. Two options:

Option 1: Reinstall Python and check "Add Python to PATH" during setup.

Option 2: Try using py instead of python:

py app.py
❌ "Address already in use" / port 5000 busy

Another program is using port 5000. Run Flask on a different port:

Edit the last line of app.py to:

app.run(debug=True, port=5001)

Then open http://127.0.0.1:5001 in the browser.

On Mac, port 5000 may be used by AirPlay. Edit the last line of app.py:

app.run(debug=True, port=5001)

Then open http://127.0.0.1:5001 in the browser.

❌ Page shows "TemplateNotFound" error

Flask cannot find the HTML files. Make sure:

  • Your HTML files are inside a folder named exactly templates (lowercase)
  • The templates folder is in the same folder as app.py
flight_booking/
  ├── app.py
  └── templates/  # ← must be here, next to app.py
      ├── search.html
      ├── flight.html
      └── success.html
❌ "No flight found" even for a valid city

Check your spelling. The app does a case-insensitive match, but the city name must match exactly.

Valid cities: Delhi, Mumbai, Bangalore, Chennai, Kolkata, Hyderabad, Pune, Ahmedabad
